# Security Policy

## Supported Versions

Currently, the `main` branch of OISS is the only supported version for security updates.

## Reporting a Vulnerability

Security is a top priority for OISS, especially concerning IP masking and anonymity. 

If you discover any security vulnerability in OISS, please DO NOT open a public issue. Doing so might put users at risk.

Instead, please email the maintainer directly at: `security@oiss.example.com` (update this email before public launch).

We will try to acknowledge your report within 48 hours and work on a fix immediately.

### What to include in your report:
- A description of the vulnerability.
- Steps to reproduce it.
- Possible impact (e.g., Does it leak IP addresses? Does it bypass rate limits?).
